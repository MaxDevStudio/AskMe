# Архітектура та структура проєкту "AskMe"

Цей документ описує поточну та плановану структуру файлів проєкту, а також ключові архітектурні рішення.

## 1. Поточна структура (v1.x)

На поточному етапі проєкт є простим CLI-інструментом з основною логікою у виконуваних файлах. Це дозволяє швидко виконувати команди безпосередньо з терміналу.

```bash
AskMe/
├── architecture/
│   ├── EN_PROJECT_STRUCTURE.md
│   └── UA_PROJECT_STRUCTURE.md
├── bin/
│   ├── ask.js              # < Головний скрипт для запитів до AI
│   └── list-models.js      # < Утиліта для перегляду доступних моделей
├── docs/
│   ├── en/
│   ├── ua/
│   ├── VISION_EN.md
│   └── VISION_UA.md
├── scripts/
│   ├── hello-system.sh
│   └── status.sh
├── .env                    # < Ключі API та конфігурація
├── .gitignore
├── package.json
└── README.md
```

## 2. Планована структура (v2.x - з GUI та API)

У майбутньому планується розширення проєкту до повноцінного додатка з веб-інтерфейсом (GUI) та REST API для керування. Це вимагатиме рефакторингу та переходу до більш модульної структури.

Основна ідея — винести ядро логіки (взаємодію з AI) в окремий модуль, який буде використовуватися як CLI, так і API.

```bash
AskMe/
├── architecture/
│   └── ... (документація архітектури)
├── docs/
│   └── ... (документація проєкту)
├── public/                 # < Статичні файли для веб-інтерфейсу (GUI)
│   ├── index.html
│   ├── styles.css
│   └── main.js
├── src/                    # < Основний вихідний код
│   ├── cli/
│   │   └── index.js        # < Точка входу для CLI (замінить /bin/ask.js)
│   ├── api/
│   │   ├── server.js       # < Головний файл API (Express/Fastify)
│   │   └── routes.js       # < Маршрутизація API (/api/ask, /api/status)
│   └── core/
│       └── gemini.js       # < Ізольований модуль для взаємодії з Gemini API
├── scripts/
│   └── ... (допоміжні скрипти)
├── .env
├── package.json
└── index.js                # < Головна точка входу (запускає сервер API)
```

### Ключові зміни та обґрунтування:

1.  **Директорія `src`:** Весь основний код переноситься сюди. Це стандартна практика, що відокремлює вихідний код від конфігурації, документації та інших файлів.
2.  **Модуль `src/core/gemini.js`:** Логіка взаємодії з Gemini API виноситься в окремий, перевикористовуваний модуль. Це дозволить і CLI (`src/cli/index.js`), і API (`src/api/server.js`) використовувати ту саму функціональність, уникаючи дублювання коду.
3.  **Розділення `cli`, `api`, `public`:**
    *   `src/cli`: Код, що відповідає виключно за роботу в терміналі.
    *   `src/api`: Код для REST API, який буде основою для веб-інтерфейсу та потенційної інтеграції з іншими сервісами.
    *   `public`: Файли фронтенду (HTML, CSS, клієнтський JS) для простого GUI.
4.  **Головний файл `index.js`:** Замість кількох точок входу в `bin/`, буде один головний файл, який, наприклад, запускатиме веб-сервер. CLI-команди будуть налаштовані в `package.json` в секції `bin`.

Ця структура є більш масштабованою та підготовленою до майбутнього розвитку проєкту як "оркестратора".
