**Роль:** Інтелектуальний CLI-Диригент  
**Статус:** Активна розробка (Open Source)  
**Ліцензія:** MIT  
**Репозиторій:** [github.com/MaxDevStudio/AskMe](https://github.com/MaxDevStudio/AskMe)

## 1. Глобальна візія
AskMe Free — це легкий оркестратор для Linux-систем, що виступає інтелектуальним шаром між користувачем, операційною системою та моделями ШІ. Основна мета — зробити управління системою через термінал максимально доступним та автоматизованим.

## 2. Ключовий функціонал (v1.4.0)

### 2.1. Аварійний режим (GRUB Terminal Mode)
* **Реалізація:** Інтеграція в завантажувач через `/etc/grub.d/40_custom`.
* **Мета:** Створення «аварійного парашута». Якщо графічна оболонка (GNOME/KDE) зависає, користувач може завантажитись у чистий TTY (Runlevel 3), де AskMe готовий допомогти виправити систему.

### 2.2. Розумний пейджер (Integration with `less`)
* **Мета:** Вирішення проблеми переповнення екрана в CLI.
* **Функція:** Довгі відповіді автоматично передаються в пейджер `less` для зручної прокрутки та пошуку.
* **Логування:** Можливість миттєвого збереження вмісту у файл командою `s` прямо всередині `less`.

### 2.3. Режим розробника (JSON Support)
* **Функція:** Додавання прапорця `--json` у CLI.
* **Мета:** Надання структурованих даних для того, щоб спільнота могла будувати власні GUI, віджети або плагіни на базі нашого двигуна.

### 2.4. Локалізація та інтерфейс
* **TTY Localization:** Підтримка безшовного перемикання мов (Alt+Shift) у терміналі для спілкування з ШІ українською мовою.

## 3. Технічний стек
* **Runtime:** Node.js (забезпечує роботу майже на всіх дистрибутивах Linux).
* **AI Engine:** Google Gemini API (Flash/Pro).
* **Віртуалізація:** Підтримка роботи в ізольованих OCI-контейнерах через Toolbx/Distrobox для безпечного тестування коду.

## 4. Стратегічне розмежування
* **AskMe Free:** Відкритий інструмент для сисадмінів та спільноти (JS/Node.js).
* **AIO (AI Orchestrator):** Корпоративна Pro-версія на базі Rust для глобального управління інфраструктурою.
